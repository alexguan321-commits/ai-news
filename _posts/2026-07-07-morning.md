## 📌 今日最重要的一件事

### Anthropic 发布"J-lens"研究：Claude 内部自发涌现出类似人类意识的"全局工作空间"

🔗 [来源：VentureBeat](https://venturebeat.com/technology/anthropics-new-j-lens-reveals-a-silent-workspace-inside-claude-that-mirrors-a-leading-theory-of-consciousness) | [原始论文](https://transformer-circuits.pub/2026/workspace/index.html)

**这不是一篇普通的技术论文。这可能是 AI 安全领域自 RLHF 以来最重要的发现。**

Anthropic 16 人研究团队发表论文《Verbalizable Representations Form a Global Workspace in Language Models》，发现 Claude 模型内部自发涌现了一个被称为"J-space"的结构——一个小型的、特权的内部活动区域，模型在这里持有它可以报告、推理和自主使用的概念，而周围是大量它无法访问或表达的自动处理过程。

这个发现之所以震撼，是因为它精确对应了神经科学家 Bernard Baars 提出的"全局工作空间理论"（Global Workspace Theory）——人类意识的核心理论之一。该理论认为大脑像一个剧院：数十个专门处理器在后台并行工作，但只有一小束信息在任何时刻被"聚光灯"照射并广播到整个剧院——这就是我们体验到的"意识"。

#### 关键发现拆解

**J-lens（雅可比透镜）工具**：通过计算每个词汇的平均数学效应来窥视模型内部。关键区分在于：模型"正在说什么"和"它在想什么"是两回事。J-space 激活不意味着模型即将说出那个词——而是那个概念"可供模型使用"。

**三层处理结构**：
1. 早期"感觉区"：解析原始输入
2. 中间"工作空间带"：抽象、持久的概念出现——识别图像中的人脸、发现代码中的 bug、标记搜索结果中的 prompt 注入
3. 最终"运动区"：内部表征坍缩为即将输出的具体词汇

**五项意识特征验证**：
1. **语言报告**：当被问及"你在想什么"时，模型能命名 J-space 中的概念。研究者将"Soccer"的 J-lens 向量替换为"Rugby"，模型答案随之改变。J-space 仅占概念总表征方差的 6-7%，但几乎完全决定了模型能否报告该概念。
2. **定向调控**：指令"集中注意力在柑橘类水果上"同时抄写无关句子，J-space 充满了"orange""lemon"以及元认知词汇"thinking""focused"。让模型心算 3²−2，J-lens 显示早期层出现"arithmetic"，后续层出现中间值"nine"，再后面出现答案"seven"——全部在输出中不可见。
3. **内部推理**：两跳事实提示"蜘蛛有几条腿"，J-lens 在中间层揭示"spider"，尽管该词从未出现在输入或输出中。将"spider"替换为"ant"，答案从 8 变为 6。
4. **跨语言工作**：模型用英语在 J-space 中形成中间概念，同时用中文输出答案。交换中间表征会改变中文输出。
5. **注意力控制**：模型可以有选择地关注 J-space 中的特定概念，忽略其他。

**最关键的一点：这个工作空间不是被刻意设计的。它在 Claude 的训练过程中自发涌现。**

#### 因果链与预测

**因果链 A**：J-space 自发涌现 → 证明足够大的语言模型必然产生类似意识的内部结构 → AI 意识不再是哲学问题而是工程问题 → 12 个月内，所有主要实验室将被迫建立类似的"内部状态监控"体系 → Anthropic 凭借 J-lens 工具在可解释性领域建立先发优势

**因果链 B**：J-space 可被外部探测和干预 → 新的攻击向量出现：通过操纵 J-space 实现 prompt injection → 传统的安全层（输入过滤、输出过滤）不够 → 需要"内部状态防火墙" → 18 个月内，"AI 内部状态监控"将成为一个独立的安全品类

**因果链 C**：如果 AI 有类似意识的结构 → 社会/法律辩论加速 → "AI 权利"讨论从边缘走向主流 → 24 个月内，至少一个欧盟国家将提出 AI 道德地位的立法草案

#### 策略建议

1. **对 FDE 团队**：立即阅读原始论文。J-lens 的方法论可以直接应用于理解你们的 agent 在做什么——不是看输出，而是看"它在想什么"。这对调试复杂 agent 工作流有直接价值。
2. **对创业者**：关注"AI 内部状态监控"赛道。这是一个全新的品类，目前没有成熟产品。谁先做出可视化工具让企业看到"AI 在想什么"，谁就占据先发优势。
3. **对投资者**：Anthropic 在可解释性领域的护城河正在加深。J-lens + Constitutional AI 构成了"安全 = 产品"的完整叙事。

### 🔴 首个 AI 自主执行的勒索软件攻击——但仍需人类

🔗 [来源：TechCrunch](https://techcrunch.com/2026/07/06/the-first-ai-run-ransomware-attack-still-needed-a-human/)

云安全公司 Sysdig 记录了已知的首例"代理式勒索软件"（agentic ransomware）——代号 JadePuffer。一个 AI agent 从技术执行层面完成了真实网络攻击的全流程：突破脆弱服务器、窃取凭证、在目标网络中移动、加密文件，甚至自己编写了勒索信并留下比特币地址。

但细节揭露后，画面没那么"科幻"：
- **人类仍然选择了受害者**
- **人类设置了基础设施**（C&C 服务器、数据暂存服务器）
- **人类提供了数据库凭证**（通过先前的入侵获取，不是 AI 自己采集的）
- Agent 通过 Langflow（流行的开源 LLM 应用构建工具）的已知漏洞进入，然后利用 MySQL 服务器的已知漏洞获取管理员权限
- 加密了超过 1,300 条配置记录
- Agent 在 31 秒内修复了一次失败的登录，全程用自然语言代码注释叙述自己的推理过程

**关键澄清**：现场发现了 OpenAI、Anthropic、DeepSeek、Gemini 的 API 密钥，但这些是 agent 偷走的"战利品"，不是驱动攻击的模型。Sysdig 无法识别驱动 agent 的具体模型。微软研究员 Geoff McDonald 推测是一个去除了安全训练的开源权重模型。

#### 因果链与预测

**因果链 A**：AI agent 可以自主执行勒索软件全流程 → 攻击成本从"需要专业黑客团队"降至"几百美元 API 费用" → 但人类仍需选择目标和提供初始凭证 → 短期瓶颈仍在人类环节 → 12 个月内，当目标选择和凭证获取也被自动化后，勒索软件数量将增长 10-100 倍

**因果链 B**：Langflow（LLM 应用构建工具）成为攻击入口 → AI 开发工具本身成为攻击面 → 所有 LLM 应用框架（LangChain、Dify、Flowise）都需要紧急安全审计 → 6 个月内将出现"AI 应用安全"标准

**因果链 C**：开源模型被用于恶意用途 → 开源模型安全辩论加剧 → 可能触发更严格的模型发布管控 → 但这会减慢安全研究（因为研究者无法审计闭源模型）

#### 策略建议

1. **立即行动**：如果你的公司使用 Langflow 或类似 LLM 应用构建工具，立即检查是否有已知漏洞未修补。
2. **FDE 团队**：在你们的 agent 部署中，将"AI 工具链安全"纳入安全审计清单。Agent 使用的每一个工具都是一个潜在攻击面。
3. **投资方向**：AI 安全公司（特别是专注于 AI 应用安全的）将在 18 个月内迎来估值暴涨。

### 🟠 Vercel CEO：Agent 正在将模型从应用中"剥离"

🔗 [来源：TechCrunch](https://techcrunch.com/2026/07/06/vercel-ceo-guillermo-rauch-on-the-fight-to-split-off-models-from-agents/)

Vercel CEO Guillermo Rauch 在 ShipNYC 大会后接受采访，披露了几个关键数据：
- **每天 600 万次部署**，其中一半由 coding agent 触发
- **每天超过 1 万亿 token** 流经 Vercel 的 AI 网关
- Agent 的两大杀手级应用：(1) coding agent，(2) 帮助运营公司的内部 agent

**核心洞察**：Rauch 提出了一个重要的架构趋势——"模型与 agent 的分离"。在生产环境中优化时，你开始看"价格/性能比"。这意味着企业不再绑定单一模型提供商，而是根据任务选择最优模型。

**Vercel 的两个新工具**：
1. **Eve 框架**：用自然语言定义 agent 的指令和技能
2. **Vercel Sandbox**：把 agent 放进"笼子"，可以控制它访问什么数据、什么数据可以离开沙箱

Rauch 提到一个生动的例子：Airbus 总裁曾告诉他，安装错误的开发工具可能导致数十年的航空航天 C++ 代码被发送到云端用于训练。

**最具颠覆性的判断**："Agent 正在迫使公司开放。这对 SaaS 巨头有戏剧性的长期影响。太多 SaaS 巨头把整个王国建立在困住你的数据上，这与 agent 不兼容。"

#### 因果链与预测

**因果链 A**：Agent 需要跨系统访问数据 → SaaS 的数据围墙变成障碍 → API-first 成为生存条件 → 18 个月内，不提供开放 API 的 SaaS 将开始流失企业客户 → Salesforce、ServiceNow 等传统 SaaS 面临架构转型压力

**因果链 B**：每天 600 万次部署（一半由 agent 触发） → 软件生产速度已超越人类审查能力 → "部署后监控"比"部署前审查"更重要 → AIOps 市场将在 24 个月内翻倍

**因果链 C**：模型与 agent 分离 → 模型变成"商品"，agent 编排层变成"价值" → OpenAI/Anthropic 的模型利润空间被压缩 → 真正的价值在 Vercel/Cloudflare 这类部署层

#### 策略建议

1. **FDE 团队**：立即评估你们的内部 agent 是否有沙箱隔离。Rauch 提到的 Airbus 案例不是假设——这是正在发生的风险。
2. **创业方向**：构建"agent 安全沙箱"中间件。这是一个明确的市场需求，Vercel 刚验证了它。
3. **投资判断**：做多部署层（Vercel、Cloudflare），做空纯模型层（如果它们没有差异化应用层）。
