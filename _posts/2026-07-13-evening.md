## 📌 今日最重要的一件事

### 🔴 OpenAI 将 Codex、ChatGPT、Work 三合一，打造桌面"Work OS"

🔗 [来源：ZDNet](https://www.zdnet.com/article/openai-gutted-chatgpt-desktop-app-for-codex-work/) | [来源：TechTimes](https://www.techtimes.com/articles/320087/20260710/chatgpt-work-free-every-plan-what-openais-codex-merger-changes-you.htm) | [来源：WindowsForum](https://windowsforum.com/threads/codex-becomes-openais-desktop-work-os-chatgpt-and-agents-converge-in-2026.434874)

**发生了什么：** 7月9日，OpenAI 正式宣布将三个独立产品——Codex（编码 Agent）、ChatGPT（对话）、Work（文档/表格/应用生成 Agent）——合并为**一个统一的桌面应用**，覆盖 macOS 和 Windows，所有计划（包括免费版）均可使用。

**核心架构变化：**
- **三模式统一入口**：Chat / Work / Codex 三个模式共享项目、文件、浏览器访问权限、统一插件目录和治理层
- **Work 模式免费开放**：所有计划（含 Free）都能使用 Work 模式——这是 OpenAI 首次将 Agent 级生产力工具下放给免费用户
- **Codex 不退场**：Codex 作为"专业编码体验"保留在桌面应用内，而非被吞并消失

**为什么这是今天最重要的事：**

这不是一个简单的产品合并。这是 OpenAI 在**重新定义 AI 产品的边界**——从"聊天工具"变成"工作操作系统"。

**因果链分析：**

```
OpenAI 三合一 → 用户不再需要切换工具（Chat 聊天 → Work 出文档 → Codex 写代码）
→ 用户工作流被锁定在 OpenAI 生态内 → 切换成本指数级上升
→ Microsoft Copilot 面临直接竞争（同样是桌面集成，但 Copilot 依赖 Office 生态）
→ Google Gemini 必须加速从搜索延伸到桌面生产力
→ AI 产品竞争从"模型能力"转向"工作流锁定"
```

**大胆预判：**

1. **12个月内**，OpenAI 桌面应用 DAU 将突破 5000万，其中 Work 模式贡献 30%+ 日活。因为免费 + 三合一 = 无脑替换 Notion + ChatGPT 网页版。
2. **18个月内**，Microsoft 将被迫将 Copilot 从 Office 插件升级为独立"Work OS"，否则企业用户会大规模迁移到 OpenAI 桌面端。因果链：OpenAI 免费 Work → 个人用户习惯形成 → 带入企业 → 企业 IT 部门发现 OpenAI 比 Copilot 便宜且更灵活 → Copilot 付费率下降。
3. **6个月内**，ChatGPT 桌面端将取代 Notion AI 成为个人知识管理的首选入口。因果链：Work 模式能直接生成文档/表格/应用 → 用户不再需要 Notion 做中间层 → Notion AI 的差异化消失。

**策略建议：**
- **个人用户**：立即迁移到新版 ChatGPT 桌面端，把 Work 模式作为你的默认文档工具。Notion AI 的订阅可以在 Q4 前取消。
- **开发者**：关注统一插件目录的 API——这是新的分发渠道。第一批上架的插件将获得巨大流量红利。
- **企业决策者**：在 Q3 结束前评估 OpenAI 桌面端 vs Copilot 的 TCO（总拥有成本）。OpenAI 免费版已经包含了 Work 能力，这改变了 ROI 计算。

### 🟠 DeepSeek V4 正式版本周发布，首次引入峰谷定价

```

→ 1M 上下文成为标配 → 长文档/代码库级别的 Agent 应用成为可能
→ 与 OpenAI 三合一桌面端形成"开源 vs 闭源"的正面竞争
```

**策略建议：**

- **架构师**：1M 上下文意味着你可以把整个代码库放进上下文窗口。重新评估你的 RAG 架构——可能不再需要向量数据库做第一层检索。

### 🟠 HeyGen 开源 HyperFrames：用 HTML 写视频，AI Agent 的视频渲染引擎

🔗 [来源：GitHub](https://github.com/heygen-com/hyperframes) | [来源：HeyGen Research](https://www.heygen.com/research/html-to-video) | [来源：HeyGen Docs](https://developers.heygen.com/hyperframes-overview)

**发生了什么：** HeyGen 开源了 HyperFrames——一个将 HTML/CSS/JS 组合直接渲染为视频（MP4/MOV/WebM/GIF/PNG 序列）的框架。核心理念：**Write HTML, Render Video. Built for Agents.**

**技术亮点：**
- **前端优先路径**：AI Agent 擅长生成 HTML/CSS/JS，HyperFrames 把这个能力直接转化为视频生成能力
- **确定性渲染**：无头浏览器 + 帧捕获，确保每次渲染像素级一致
- **支持本地和云端渲染**：可通过 HeyGen Cloud Rendering API 大规模渲染
- **AI Agent 友好**：不需要视频编辑知识，Agent 写 HTML 就能生成产品 demo、动画解说、个性化视频

**因果链分析：**

```
HyperFrames 开源 → AI Agent 可以用 HTML 生成视频（而非依赖不可控的 AI 视频模型）
→ 视频生成从"创意工作"变成"代码工作" → 可编程、可参数化、可大规模
→ 个性化视频营销成本降至接近零（每个用户看到不同的视频，但都是代码生成的）
→ 传统视频编辑工具（Premiere、After Effects）面临降维打击
→ AI 短视频/产品 demo/动画解说成为 Agent 的标准输出格式
→ HeyGen 通过 Cloud Rendering API 获得基础设施收入
```

**大胆预判：**

1. **6个月内**，至少 3 个 AI Agent 框架（LangChain、CrewAI、AutoGen）将集成 HyperFrames 作为默认视频输出工具。因果链：开源 + Agent 友好 + 确定性渲染 = 完美匹配 Agent 工作流。
2. **12个月内**，"HTML 视频"将成为一个新的开发者品类，GitHub 上出现 100+ 个基于 HyperFrames 的视频模板库。
3. **18个月内**，HeyGen 的 Cloud Rendering 收入将超过其数字人视频收入。因果链：HyperFrames 开源 → 开发者涌入 → 云端渲染需求爆发 → 基础设施收入 > 内容收入。

**策略建议：**
- **开发者**：本周 clone HyperFrames，尝试用你的 AI Agent 生成一个产品 demo 视频。你会发现这比传统视频制作快 100倍。
- **营销团队**：评估"个性化视频"的可行性——每个客户收到一个用他们名字和公司数据生成的 demo 视频，成本几乎为零。
- **投资者**：HeyGen 的估值逻辑正在从"数字人 SaaS"转向"AI 视频基础设施"。后者是更大的故事。

## 🐙 GitHub Trending

### 1. 🌟 heygen-com/hyperframes — HTML to Video for AI Agents
🔗 [来源：GitHub](https://github.com/heygen-com/hyperframes)

**技术亮点：** 把 HTML/CSS/JS 组合渲染为视频。核心创新是"确定性渲染"——无头浏览器逐帧捕获，确保每次输出像素级一致。这对 AI Agent 至关重要：Agent 生成的视频必须可复现、可调试、可参数化。传统 AI 视频模型（Sora、Veo）做不到这一点。
