## 📌 今日最重要的一件事

### 🔴 AI PPT 生成终于"能用"了——ppt-master 单日冲上 GitHub Trending 第一

**事实：**
- Hugo He 的开源项目 [ppt-master](https://github.com/hugohe3/ppt-master) 今日登顶 GitHub Trending，**32,688 stars / 2,797 forks**
- 核心能力：从任意文档生成**真正可编辑的 PowerPoint**——原生形状（非图片）、动画、Speaker Notes 自动配语音旁白、支持自定义 .pptx 模板
- 创建于 2025-12-10，半年内从零到 3.2 万星，增长曲线陡峭

**判断：**
这不是又一个"AI 做 PPT"的 demo。ppt-master 解决的是这个赛道一直以来的核心痛点：**生成的 PPT 不可编辑、不可复用、不可定制**。之前的方案（Gamma、Beautiful.ai、Tome）本质上是生成"好看的图片墙"，用户拿到后还是要重做。ppt-master 输出的是原生 PowerPoint 元素——形状、文本框、动画轨道——这意味着企业用户可以把它嵌入自己的模板体系，而不是被锁定在某个 SaaS 平台里。

**为什么现在爆发：**
- 多模态 LLM 的文档理解能力终于够用（能解析复杂文档结构）
- python-pptx 等库的成熟让"程序化生成原生 PPT"变得可行
- 企业用户对"AI 生成但不可编辑"的容忍度已经耗尽——市场在等一个真正能用的方案

**So What — 对 A 哥意味着什么：**
1. **宝洁内部每周成百上千份 PPT。** 季度业务回顾、品牌提案、供应链报告——如果 FDE 团队能把 ppt-master 接入内部文档流水线（比如从 Confluence/飞书文档自动生成品牌标准 PPT），**节省的不是几分钟，是组织级别的重复劳动**。
2. **"支持自定义模板"是关键。** 宝洁有严格的品牌视觉规范。ppt-master 的模板跟随能力意味着生成的 PPT 可以直接套用宝洁官方模板——这是企业落地的最后一百米。
3. **FDE 团队可以本周就启动 POC。** Python 项目、开源、活跃维护——技术风险低，业务价值高。

### 2. 🟠 Vibe-Trading：港大团队开源多 Agent 交易系统
**事实：** [Vibe-Trading](https://github.com/HKUDS/Vibe-Trading) 是香港大学数据科学团队开源的个人交易 Agent，支持多 Agent 协作、MCP 协议、LLM 驱动的策略生成与回测。13,491 stars / 2,573 forks。

**判断：** "Vibe Coding"之后是"Vibe Trading"——这个命名不是巧合。它反映的是一个趋势：**AI Agent 正在从"辅助决策"走向"自主执行"**。交易场景是 Agent 自主执行的理想试验场——数据实时、反馈即时、结果可量化。Vibe-Trading 的技术架构（多 Agent + MCP）值得关注，因为它展示了一种可复用的 Agent 协作模式：**一个 Agent 负责数据分析，一个负责策略生成，一个负责风控，一个负责执行**——这种模式完全可以迁移到供应链决策、库存管理、需求预测等宝洁核心场景。

**对 A 哥的启示：** 不要只看"交易"这个应用场景，要看**架构模式**。FDE 团队在设计供应链 Agent 时，可以参考 Vibe-Trading 的多 Agent 分工模式——让不同 Agent 各司其职，通过 MCP 协议协作，而不是做一个"什么都会但什么都不精"的万能 Agent。

## 🔬 深度分析

### "Vibe X" 模式的本质：AI Agent 的自主执行光谱

从 Vibe Coding → Vibe Trading，我们看到一个模式正在形成：**"Vibe X" = 人类定义目标 + AI Agent 自主执行过程**。

这个模式的核心特征是：
1. **人类的角色从"操作者"变成"审核者"** — 你不再写代码/做交易，你审核 AI 写的代码/做的交易
2. **反馈循环必须极短** — 代码可以立刻跑，交易可以立刻验证，所以 Agent 能快速迭代
3. **可回滚性是前提** — 之所以敢让 Agent 自主执行，是因为 git 可以 revert，仓位可以平仓

**对宝洁的启示：** 供应链决策是否满足"Vibe X"的条件？
- 反馈循环：供应链决策的反馈周期是周/月级别，比代码/交易慢得多
- 可回滚性：库存决策一旦执行，回滚成本很高
- 结论：**供应链不适合完全的"Vibe X"，但适合"Vibe X with Human-in-the-Loop"** — Agent 生成方案，人类审批执行。FDE 团队应该设计的是这种"半自主"模式，而不是追求全自动。
