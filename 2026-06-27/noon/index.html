🧠 Ontology & Knowledge Engineering

🟠 "Vector RAG Isn't Enough — I Built a Context Graph Layer for Multi-Agent Memory"（Towards Data Science）

作者在同一多 Agent 对话上 benchmark 了三种方案：原始聊天记录、纯向量 RAG、上下文图谱。结论：向量检索在关系推理上有明显弱点，Context Graph 层是必要的。

判断： 这验证了 Ontology/Knowledge Graph 在 Agent 时代的价值。纯向量搜索只能做"相似度匹配"，做不了"关系推理"。当 Agent 需要理解实体之间的关系（供应商→产品→仓库→运输路线），Knowledge Graph 是不可替代的。

对 A 哥的启示： 宝洁供应链的知识体系（供应商、SKU、仓库、运输路线、合规要求）天然是图谱结构。你的 PD One Platform 如果只做向量 RAG，会在复杂查询上碰壁。建议在供应链知识管理中引入 Knowledge Graph 层。

🔬 深度分析

主题一：AI 模型的"武器化"与"许可化"——全球 AI 竞争格局正在重塑

今天最重要的两条新闻看似独立，实则指向同一趋势：

  • Anthropic Mythos 需要白宫批准才能向"select organizations"开放
  • OpenAI GPT-5.6 被白宫要求延迟发布

这不是传统的"监管"。传统监管是"你不能做什么"；这是"你必须经过许可才能发布"。AI 模型正在被当作军事技术来管理。

深层逻辑： 1. 美国政府意识到前沿 AI 的能力已经触及国安红线（网络攻击、生物武器设计、自主决策） 2. 中美 AI 竞争不允许任何"能力外溢"——即使是商业发布也可能被中国公司通过蒸馏获取 3. 这创造了一个新的"AI 许可市场"——谁能拿到许可，谁就有竞争优势

对全球 AI 格局的影响：

  • 美国公司： 发布节奏被政治化，商业计划被打乱
  • 中国公司： 加速自主可控。GLM 5.2、Qwen 系列不再只是"替代品"，而是"战略必需品"
  • 欧洲： 彻底绝望，开始认真搞 AI 主权。但时间窗口已经很窄
  • 企业用户： 不能押注单一供应商。模型可插拔不是技术选择，是生存策略

对 A 哥创业方向的具体建议： 1. "AI 模型合规中间件" 是一个正在形成的需求。帮企业做模型切换、合规审查、数据隔离的解决方案 2. 在宝洁内部推动"多模型架构"：Dify 工作流中同时接入 GPT/Claude/Qwen/GLM，根据场景和合规要求动态切换 3. 创业创收方向： 帮跨国企业做"中国 AI 模型合规评估"——哪些中国模型可以在全球场景使用，哪些有 IP 风险

主题二：机器人行业的"价格击穿"——从 Demo 到 ROI 的临界点

今天机器人新闻的密度和信号强度是近期最高的：

  • Unitree R1: $4,900 现货

算一笔账：

模式： 这些看似不相关的事件指向同一个方向：Agent 的基础设施层正在从"实验性"变成"生产级"。 世界模型、专用 LLM、工具协议、记忆系统、安全框架——这些是 Agent 大规模部署的必要条件，它们正在同时成熟。

判断： 2026 下半年到 2027 上半年，会出现第一批"生产级 Agent 平台"。不是 demo，不是 POC，而是真正在企业中 7×24 运行的 Agent 系统。

行动建议： A 哥的 Dify + FDE 团队正在做 Agent 落地，方向完全正确。关键是要在基础设施层做对选择： MCP 作为工具协议、Knowledge Graph 作为记忆层、多模型架构作为推理层。现在做对架构选择，半年后就能跑赢竞争对手。

洞察二：Vibe Coding 的反噬正在形成行业共识

现象： Hacker News 上关于 vibe coding 的讨论密度极高，且标题几乎全是负面的：

  • "After two years of vibecoding, I'm back to writing by hand" (865 points)
  • "The cult of vibe coding is dogfooding run amok" (616 points)
  • "Breaking the spell of vibe coding" (434 points)
  • "Vibe coding kills open source" (330 points)
  • "Will vibe coding end like the maker movement?" (405 points)

模式： 这些高赞帖子的核心论点是：vibe coding 降低了入门门槛，但提高了维护成本。 AI 生成的代码看起来能跑，但缺乏可维护性、可审计性、安全性。开源社区正在被低质量的 AI 生成代码淹没。

判断： "Vibe Coding 幻灭期"已经到来。行业正在从"AI 写代码好神奇"转向"AI 写的代码谁来维护"。这会催生新的工具和方法论：AI 代码审计、AI 生成代码的可维护性标准、Human-in-the-Loop 的代码审查流程。

行动建议： 1. 在宝洁 FDE 团队中建立"AI 代码质量标准"：不是不让用 AI 写代码，而是 AI 生成的代码必须经过审查、测试、文档化 2. 创业机会： "AI 代码审计工具"或"AI 代码质量认证"是一个正在形成的需求 3. 对 FDE 团队的定位调整： FDE 不只是"用 AI 工具的人"，而是"确保 AI 生成代码质量的人"。这是 FDE 区别于普通 vibe coder 的核心价值

洞察三：推理成本下降 + Agent 基础设施成熟 = FDE 的黄金时代

现象：

  • OpenAI/Broadcom 推理芯片（降本）
  • Blackstone $300 亿亚太算力投资（供给增加）
  • Qualcomm 收购 Modular（边缘推理增强）
  • MCP/Agent SDK/Context Graph 成熟（基础设施就绪）
  • GLM 5.2/Qwen 开源模型追赶（选择增多）

模式： 推理成本下降 + Agent 基础设施成熟 + 模型选择增多 = Agent 部署的经济模型和技术条件同时成立。

判断： 这意味着 FDE（Forward Deployed Engineer）这个角色正在从"内部布道者"变成"核心业务角色"。当 Agent 可以在企业中以低成本、高可靠性运行时，谁能把 Agent 落地到具体业务场景，谁就是企业的核心竞争力。

行动建议： 1. A 哥的 FDE 团队应该重新定义自己的价值主张： 不是"我们会用 AI 工具"，而是"我们能以 ROI 可计算的方式把 Agent 部署到业务场景" 2. 建立 FDE 的"Agent 落地方法论"： 场景评估 → ROI 计算 → 架构设计 → POC → 规模化。这套方法论本身就是可以对外输出的产品 3. 创业方向： "FDE as a Service"——帮企业做 Agent 落地的咨询服务。A 哥在宝洁的 FDE 实践经验，本身就是一份有价值的资产
