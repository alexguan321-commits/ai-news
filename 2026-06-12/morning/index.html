📌 今日最重要的一件事

DIRECT：Embodied Planner 的 Test-Time Compute 该花在哪里？ 🟠

来源：arXiv 2606.12402

发生了什么： UC Berkeley 的 Jadelynn Dao 等人发表论文，系统性地回答了 VLM-as-Planner 领域一个关键但未被严谨研究的问题：当我们给 embodied agent 的 planner 增加 test-time compute（更多 rollout、更深搜索）时，收益是均匀分布的吗？答案是否定的——收益高度不均匀，且常常边际递减。他们提出 DIRECT 框架，动态决定何时以及何地分配额外的 test-time compute。

为什么现在发生： 2025-2026 年，"scale test-time compute" 成为 LLM/Agent 领域的显学（OpenAI o-series、Anthropic thinking tokens、各种 search-based agent）。但大家默认"更多 compute = 更好结果"，很少有人去严格测量 embodied setting 下的实际收益曲线。这篇论文填补了这个空白。

So What for Alex：

  • 创业视角： 如果你做 AI agent SaaS，"智能 compute 分配"是一个可以产品化的差异化点——同样的任务，花 1/3 的 token 达到同样效果，就是竞争力。
  • 行动建议： 读 paper 的 Section 4（收益分布分析），把它的方法论应用到你现在跑的 Dify workflow 里——哪些 node 值得加 retry/反思，哪些直接 pass-through。

🟠 APPO: Agentic Procedural Policy Optimization 来源：arXiv 2606.12384 现有 agentic RL 方法在 tool-call boundary 做 credit assignment，太粗。APPO 提出 procedural-level 的信用分配——识别 agent 多步工具调用中哪一步真正贡献了最终结果。 So What： 这是让 agent 真正学会"复杂工作流"的关键。你跑 Dify 多步 agent 时，如果未来有 RL fine-tune 需求，APPO 的思路值得跟进。

🟠 OCELOT: Inference-Leakage Budgets for Privacy-Preserving LLM Agents 来源：arXiv 2606.12341 LLM agent 代替用户读文件、调工具、调 API，每一步都可能泄露 PII。OCELOT 提出 inference-leakage budget——给 agent 的每一步设定隐私预算，跨步骤累积追踪。

🟠 Breaking Entropy Bounds: Accelerating RL Training via MTP with Rejection Sampling 来源：arXiv 2606.12370 RL rollout 是 LLM 训练的瓶颈。本文用 Multi-Token Prediction + rejection sampling 加速 rollout，突破了 entropy bound 的限制。
