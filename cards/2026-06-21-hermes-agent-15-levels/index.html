15 Levels of Hermes Agent: From Chatbot to 24/7 Autonomous System

YanXbt (@IBuzovskyi) 2026-06-21 Hermes Agent | 自主系统 | 技能 | Cron | 多Profile | Kanban | 知识管理 👁 - views

Original ↗

Why（背景动机）

大多数人安装 Hermes Agent 后只用作聊天机器人：输入提示、得到响应、关闭标签页。这只覆盖了 Agent 能力的 10%。

核心问题：如何从"我提问，它回答"进化到"它工作，我审查"？

这篇文章映射了 Hermes Agent 的 15 个使用层级，从第一次提示到无需人工干预的 24/7 自主系统。每个层级都建立在前一个之上，但可以跳到适合当前设置的任何层级。

What（核心定义）

三阶段 15 层级架构

阶段 层级 核心特征
Phase 1: Foundation 1-3 你在使用 Hermes，Agent 响应你的请求
Phase 2: Leverage 4-7 Hermes 更聪明地工作，你停止做 Agent 能处理的任务
Phase 3: Autonomy 8-15 Hermes 无需你工作，系统随时间复合增长

15 层级详解

Level 名称 核心能力
1 One-Shot Prompts 基础交互，执行工具调用
2 Memory + SOUL.md 跨会话记忆，个性化身份
3 Slash Commands /background /steer /queue /model
4 Skills + Right Model 按需加载专业知识，每个技能用最适合的模型
5 MCPs 连接外部工具（Gmail、Calendar、Notion、Slack 等）
6 Sub-Agents 并行执行，隔离上下文，多模型协作
7 Async Operations /goal 持久目标、Cron 定时任务、Checkpoint 回滚
8 Multi-Profile 独立 Profile（SOUL/config/memory/skills/cron 完全隔离）
9 Self-Improving Knowledge Base LLM Wiki（Karpathy 模式），自改进知识库
10 Kanban Orchestration SQLite 任务板，多 Profile 协作，依赖链
11 Voice Mode 语音消息跨平台（6 STT + 5 TTS 提供商）
12 Browser Automation 控制浏览器，填表、提取数据、交互 Web 应用
13 API Server OpenAI 兼容 HTTP 端点，任意前端连接
14 IDE Integration (ACP) VS Code/Zed/JetBrains 内运行，编辑器上下文
15 Profile Distributions 打包为 git repo，一键安装整个 Agent 设置

How（具体方法/参数）

Phase 1: Foundation（Level 1-3）

Level 2: Memory + SOUL.md 


# SOUL.md 示例（50-80 行）
identity: 你的身份定义
voice: 沟通风格
operations: 操作规则
restrictions: 限制条件

Level 3: Slash Commands 


/background <prompt>  # 后台任务，主会话保持自由
/steer <prompt>       # 中途注入消息，不中断当前执行
/queue <prompt>       # 排队后续任务，等当前完成后自动运行
/model <name>         # 会话中切换模型

Phase 2: Leverage（Level 4-7）

Level 4: 模型-任务匹配

任务类型 推荐模型 成本
Web 搜索 DeepSeek V4 Flash $0.10/M
代码审查 Claude Opus 4.8 $5/$25/M
内容写作 Claude Sonnet 4.6 $3/$15/M
编码（性价比） GPT-5.5 $2/$12/M
研究（带 grounding） Gemini 2.5 Pro $1.25/$10/M
批量子 Agent DeepSeek V4 $0.30/$0.50/M
/goal judge Gemini Flash 最便宜
自托管（免费） Qwen 3 8B via Ollama $0

Level 7: Async Operations

/goal 持久目标： 


/goal find 100 clinics in Toronto,
build a landing page for each,
draft personalized emails to each clinic.

Cron 三层成本模型： 1. no_agent 模式：脚本就是任务，$0 永远 2. wakeAgent 门控：脚本决定是否唤醒 LLM，$0 直到有变化 3. context_from：链接任务输出为管道，无需框架

Phase 3: Autonomy（Level 8-15）

Level 8: Multi-Profile 


hermes -p scout chat      # 信号发现
hermes -p analyst chat    # 研究综合
hermes -p briefer chat    # 简报交付

Level 9: LLM Wiki 


# config.yaml
WIKI_PATH=~/obsidian-wiki
OBSIDIAN_VAULT_PATH=~/obsidian-wiki  # 图视图

Level 10: Kanban 


/kanban create "Research 100 clinics" \
  --assignee scout --goal --goal-max-turns 15

/kanban create "Build landing pages" \
  --assignee coder --goal --goal-max-turns 20 \
  --depends-on "Research 100 clinics"

Level 15: Profile Distributions 


# distribution.yaml
distribution.yaml    # manifest
SOUL.md              # identity
config.yaml          # model and provider settings
skills/              # custom skills
cron/                # scheduled jobs
mcp.json             # connected tools

安装： 


hermes profile install github.com/user/their-agent

Token 经济学：15 层级不烧钱

控制手段 作用
Right Model Per Task 任务-模型匹配
WakeAgent Gates 无变化 = $0
No-Agent Mode 脚本即任务，$0
Pre-Run Scripts 免费收集数据
Lean Tool Sets 减少上下文大小
Tool Search 按需发现工具
Compression Threshold 提前压缩上下文
Auxiliary Models for Judge /goal judge 用便宜模型
Budget Caps 硬性限制

So What（启示和行动项）

对 A 哥当前系统的映射

文章层级 A 哥当前状态 差距
Level 1-3 ✅ 已掌握 无
Level 4 Skills ✅ 已用（知识编译、X 文章提取等） 可优化模型匹配
Level 5 MCPs ⚠️ 部分（未连接 Slack/Notion 等） 可扩展
Level 6 Sub-Agents ✅ 已用（delegate_task） 无
Level 7 Async ✅ 已用（Cron + /goal） 无
Level 8 Multi-Profile ❌ 未用（单 Profile） 可改进
Level 9 LLM Wiki ✅ 已用（核心系统） 无
Level 10 Kanban ❌ 未用 可改进
Level 11 Voice ⚠️ 部分（TTS 已配置） 可扩展 STT
Level 12 Browser ✅ 已用（网站自动化） 无
Level 13 API Server ❌ 未用 可选
Level 14 IDE (ACP) ❌ 未用 可选
Level 15 Distributions ❌ 未用 可选

立即可执行的行动

1. Multi-Profile 架构（Level 8）

  • 创建 `scout` Profile：信号发现（Cron 采集）
  • 创建 `analyst` Profile：研究综合（知识编译）
  • 创建 `briefer` Profile：简报交付（报告生成 + 推送）
  • 每个 Profile 独立 SOUL.md、memory、skills

2. Kanban 编排（Level 10）

  • 季度竞品分析作为 Kanban 项目
  • 12 张卡片：3 竞品 × 4 维度（定价、功能、定位、招聘信号）
  • 依赖链自动管理

3. 模型-任务匹配优化（Level 4）

  • Web 搜索 → DeepSeek V4 Flash（当前用 qwen3.7-plus，成本高）
  • 知识编译 → 保持 qwen3.7-plus（需要强推理）
  • /goal judge → Gemini Flash（最便宜）

对 AI 创业的核心启示

1. 从工具到系统的跃迁发生在 Level 7→10

  • Level 7 之前：Agent 是工具
  • Level 8-10：Agent 是系统（Multi-Profile + Wiki + Kanban）

2. Token 经济学是可持续性的关键

  • WakeAgent Gates：无变化 = $0
  • No-Agent Mode：脚本即任务
  • 模式：把昂贵模型的工作推给免费代码、便宜模型、压缩上下文

3. Profile Distributions 是产品化的路径

  • 你的 Agent 设置 = 可打包、可分发、可销售的产品
  • 新客户一键安装整个研究部门

关键洞察

"The jump from level 7 to level 10+ is where the agent stops being a tool and becomes a system. Multi-profile architecture, self-improving knowledge, Kanban orchestration. You review work that happened without you."

大多数人停在 Level 1-2。从 Level 3 到 7 是分钟到小时的节省。从 Level 7 到 10+ 是工具到系统的跃迁。
