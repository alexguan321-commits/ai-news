Why（背景动机）

大多数 AI 工程师知道如何构建一个 Agent，但很少有人知道如何构建一个在第一次尝试后能变得更好的系统。这个差距价值六位数。

核心洞察：当今生产环境中最强大的 AI 系统不是单次模型调用，而是循环（Loops）。Agent 是工人，Loop 是让工人进步的东西。

What（核心定义）

20 个循环设计模式，分为 5 大类：

Category 1 — 质量改进循环（Quality Loops）

让输出在离开系统前变得更好

| # | 模式 | 核心机制 | 应用场景 | |---|------|----------|----------| | 1 | Generate → Critique → Rewrite | 生成→批评→重写，直到质量达标 | 写作、代码审查、报告 | | 2 | Score-and-Retry Loop | 生成→评分→低于阈值则重试 | 可量化质量：提取准确率、格式合规 | | 3 | Multi-Critic Loop | 4个独立评审：正确性/风格/安全/领域 | 医疗AI、法律文档、金融分析 | | 4 | Adversarial Critique Loop | 批评者的唯一目标是"打破"答案 | 研究综合、投资论点、风险分析 | | 5 | Judge Ensemble Loop | 多个评审者打分取共识 | 单模型评估不可靠、高风险场景 |

Category 2 — 记忆循环（Memory Loops）

从过去学习，让下次更聪明

| # | 模式 | 核心机制 | 应用场景 | |---|------|----------|----------| | 6 | Reflexion Loop | 失败→分析原因→存储教训→带教训重试 | 最重要的自我改进模式 | | 7 | Memory Update Loop | 每次任务后存储：决策/结果/改进方案 | 长期运行的系统 | | 8 | Error Library Loop | 存储所有失败，新任务前先查错误库 | 避免重复犯同样错误 | | 9 | Success Pattern Loop | 也存储成功案例，复现成功方法 | 从胜利中学习，不只是错误 | | 10 | Memory Compression Loop | 记忆增长后压缩为更高层抽象 | 保持上下文可管理、系统快速 |

Category 3 — 规划循环（Planning Loops）

当现实变化时调整计划

| # | 模式 | 核心机制 | 应用场景 | |---|------|----------|----------| | 11 | Plan → Execute → Replan | 计划不是固定的，是螺旋式收紧 | 环境变化、长周期目标 | | 12 | Dynamic Workflow Loop | 管道根据结果决定自己的形状 | 多文档研究、自适应内容管道 | | 13 | Goal Decomposition Loop | 大目标→子目标→任务→步骤，直到可执行 | 复杂任务分解 | | 14 | Progress Evaluation Loop | 每N步停下来问：我们真的在接近目标吗？ | 长期运行的研究Agent | | 15 | Constraint Satisfaction Loop | 持续运行直到所有约束满足 | 生产系统的业务规则 |

Category 4 — 探索循环（Exploration Loops）

通过尝试多条路径找到最佳答案

| # | 模式 | 核心机制 | 应用场景 | |---|------|----------|----------| | 16 | Branch-and-Explore Loop | 同时探索多条路径，选最佳 | 内容变体、架构决策、A/B生成 | | 17 | Tree Search Loop | 深度优先的分支探索，剪枝弱节点 | 复杂推理链、多步规划 | | 18 | Debate Loop | 两个Agent、一个话题、对立立场 | 投资决策、战略规划、风险评审 |

Category 5 — 系统优化循环（System Optimization Loops）

循环改进循环本身

| # | 模式 | 核心机制 | 应用场景 | |---|------|----------|----------| | 19 | Prompt Optimization Loop | 系统自动优化prompt：测试→评分→重写→重测 | 生产管道、自动内容系统 | | 20 | Workflow Optimization Loop | 系统测量自身性能并修改自己的工作流 | 真正自我改进的系统 |

How（具体方法/参数）

核心结构

所有 20 个模式共享一个结构： 


Act → Observe → Evaluate → Adjust

关键代码模式

Score-and-Retry Loop: 


score = evaluate(output)
while score < threshold:
    output = generate(prompt)
    score = evaluate(output)
    attempts += 1
    if attempts > max_retries:
        return best_so_far

Memory Compression: 


Before: "Failed on task A because of X"
        "Failed on task B because of X"  
        "Failed on task C because of X"

After:  "Pattern: X causes failures. Always check X first."

Prompt Optimization: 


current_prompt = "Summarize this document."

for iteration in range(max_iterations):
    outputs = [run(current_prompt, doc) for doc in test_set]
    scores = [evaluate(o) for o in outputs]
    avg_score = mean(scores)
    
    if avg_score >= target:
        break
    
    failures = [o for o, s in zip(outputs, scores) if s < threshold]
    current_prompt = improve_prompt(current_prompt, failures)
    # Prompt rewrites itself based on where it fails

Constraint Satisfaction: 


while not all_constraints_satisfied(output):
    output = improve(output, unsatisfied_constraints)

constraints = [
    budget_under_limit,
    quality_above_threshold,
    latency_under_200ms,
    tone_matches_brand,
    no_hallucinations
]

Tree Search: 


root → [A, B, C]
A → [A1, A2]   # A looks promising, expand it
B → prune      # B is weak, stop here
A1 → [A1a, A1b]
A1a → solution ✓

So What（启示和行动项）

对 AI 创业的核心启示

1. Agent 是工人，Loop 是让工人进步的东西

  • 不要只构建单次调用的 Agent
  • 构建能从失败中学习的循环系统

2. 质量不是写出来的，是循环出来的

  • 生产级输出需要 Generate → Critique → Rewrite 循环
  • 分离生成者和评审者角色

3. 记忆是长期竞争力的关键

  • Error Library：避免重复犯错
  • Success Pattern：复现成功方法
  • Memory Compression：保持系统高效

4. 最好的 Prompt 不是人写的，是进化出来的

  • Prompt Optimization Loop 让系统自动改进 prompt
  • 无需人工干预，系统自我进化

5. 终极形态：循环改进循环

  • Workflow Optimization Loop 让系统重新设计自己
  • 这才是真正的自我改进系统

对 FDE 团队转型的启示

  • Reflexion Loop 可直接用于 FDE 的知识积累：每次失败→分析→存储→带教训重试
  • Error Library 是团队知识管理的核心：避免新人重复踩坑
  • Multi-Critic Loop 可用于代码审查：正确性/风格/安全/领域四维评审

立即可执行的行动

1. 本周：在 AI 资讯系统中实现 Score-and-Retry Loop，提升报告质量 2. 本月：构建 Error Library，存储所有采集/分析失败案例 3. 下季度：实现 Prompt Optimization Loop，让报告生成 prompt 自动进化

关键洞察

"The biggest shift happening in AI right now is not better models. It's moving from Prompt → Response to Generate → Evaluate → Learn → Improve."

生产级 AI 系统的秘密不是更好的模型，而是更好的循环设计。
