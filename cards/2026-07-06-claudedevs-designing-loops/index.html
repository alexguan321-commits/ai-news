Designing Loops: 别再提示你的 Coding Agent，而是设计循环

@ClaudeDevs 2026-07-06 coding-agent | loop | autonomous-agent | claude-code | codex 👁 - views

Original ↗

title: "Designing Loops: 别再提示你的 Coding Agent，而是设计循环" source: "X (Twitter)" author: "@ClaudeDevs" url: "https://x.com/claudedevs/status/2074208949205881033" date: 2026-07-06 tags: [coding-agent, loop, autonomous-agent, claude-code, codex] category: "AI工程实践"

Why（背景动机）

当前 AI coding agent 使用模式存在瓶颈：人类频繁被打断，需要不断跟进 agent 的工作进度并给出输入。这限制了同时管理多个 agent 的能力，人类成为工作流的瓶颈。

Claude Code 创始人 Boris Cherny 提出："I don't write prompts anymore, I have loops running that prompt Claude... I uninstalled my IDE, I wasn't using it."

What（核心定义）

Loop（循环） = 让 coding agent 在自验证循环中工作，而不是传统的"启动→实现→验证→继续"模式。

核心区别：

  • 传统模式：人类频繁介入，agent 每次完成小步骤都需要确认
  • Loop 模式：agent 自主工作，自我验证，只在完成目标后才回来报告

How（具体方法）

1. 使用 /goal 命令


/goal <定义目标> + <定义验证方式> + <定义停止条件>

2. 示例 Prompt


/goal Implement everything I asked for. Verify it end to end by clicking 
through the browser using the Playwright MCP. It's not acceptable to test 
the application only through integration tests. You need to actually click 
around the app. Continue like this until it works. Fix any issues if you 
encounter them then do an end to end test again. Run Codex exec and run
the review skill with Codex and make him approve it and iterate until 
Codex has approved it. When Codex has approved it, come to me and tell 
me which servers I can test it on and exactly how to test it.

3. 四种 Loop 类型

类型 触发方式 适用场景
Heartbeat 固定间隔 健康检查、状态监控
Cron 定时调度 日报、周报、定期任务
Hook 事件触发 代码提交后自动测试
Goal 目标驱动 复杂任务自主完成

4. 关键要素

  • 目标清晰：明确要完成什么
  • 验证方式：告诉 agent 如何检查自己的工作
  • 停止条件：知道何时算完成

So What（启示和行动项）

对 AI 工程实践的启示

1. 从"提示工程师"到"循环设计师"：不再写 prompt，而是设计 agent 的自主工作流 2. 人类角色转变：从执行者变为指挥官，从跟进细节变为设定目标 3. 并行能力释放：Loop 让你可以同时管理多个 agent，大幅提升产出

对第二大脑的启示

1. 知识应该主动触发决策，而不是被动等待查询 2. 假设-验证循环：知识入库后应该持续验证是否被新信息证实或证伪 3. Keeper 已经是 Loop 实践：发现问题→自愈→验证→报告

行动项

  • [ ] 将 Loop 思维应用到知识编译流程
  • [ ] 设计"知识触发器"机制
  • [ ] 建立假设-验证循环
