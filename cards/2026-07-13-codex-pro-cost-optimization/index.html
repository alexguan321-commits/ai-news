Codex Pro 成本优化：从4小时烧完到48小时不触限

Author: Rahul (@sairahul1) Date: July 13, 2026 Engagement: 33,697 views

Core Problem

You paid $200 for Codex Pro. You hit your limit in 4 hours of a 5-hour window.

Root cause: When Codex spawns subagents, it copies the parent model exactly. Set Sol to Ultra? Every subagent also runs Sol Ultra. 3 subagents on one task = 3 Sol Ultra instances burning simultaneously.

The Ultra Trap

Ultra is not a reasoning level. It's a multiplier.

Ultra spawns 4 parallel sub-subagents inside a single agent call. On a simple task: subagents spawning subagents spawning more subagents. Recursive. Uncontrolled. Ruinously expensive.

Model Coding Benchmark Cost
Sol Ultra 91.9% 3x
Sol Extra High 88.8% 1x

3.1 extra points. 3x the cost. OpenAI didn't even publish Ultra results for their two main coding benchmarks.

Rule: Avoid Ultra entirely. Full stop.

3-Model System

Sol Extra High → The Orchestrator

Plans. Architects. Delegates. Makes the hard decisions.

Extra High gives you 58/100 on benchmarks. Max gives you 59/100. One point difference. Three times cheaper.

Sol Medium → The Executor

Writes code. Fixes bugs. Runs tests. Implements the plan.

Sol Medium still beats Claude Fable 5 by 11.4 points on long-running agent workflows. At roughly a quarter of the cost.

Luna Extra High → The Scanner

File searches. Codebase exploration. Lightweight reads. Evidence gathering.

Luna at Extra High:

  • 1.3x faster than Terra
  • 2.5x cheaper than Terra
  • Same performance on lightweight tasks

Sol + Luna Routing Trick (48 hours, zero limits hit)


→ Sol Extra High writes the plan
→ Luna Extra High executes it
→ Sol Extra High reviews the output

One loop. No recursive spawning. No Ultra clones burning in the background.

Critical Config: max_depth = 1


[agents]
max_threads = 6
max_depth = 1

max_depth = 1 is critical. It stops subagents from spawning their own subagents. One level of delegation. No recursive spawning. No runaway burns.

AGENTS.md Critical Setting


Only spawn subagents when I explicitly ask you to.
Do not spawn subagents automatically.

This stops Sol from eagerly spinning up agents on every task.

3 Agent Files


# ~/.codex/agents/fast-scan.toml
name = "fast_scan"
description = "Quick searches, codebase exploration, lightweight read-only analysis."
model = "gpt-5.6-luna"
model_reasoning_effort = "extra-high"
sandbox_mode = "read-only"
developer_instructions = """
Gather evidence quickly and return a concise summary.
Do not edit files.
"""

# ~/.codex/agents/routine-worker.toml
name = "routine_worker"
description = "Routine coding, tests, documentation, and bounded fixes."
model = "gpt-5.6-sol"
model_reasoning_effort = "medium"
developer_instructions = """
Implement the assigned bounded task and verify the result.
"""

# ~/.codex/agents/deep-worker.toml
name = "deep_worker"
description = "Difficult debugging, architecture, security, and ambiguous multi-step work."
model = "gpt-5.6-sol"
model_reasoning_effort = "extra-high"
developer_instructions = """
Handle complex work carefully, validate assumptions,
and provide strong verification.
"""

Effort Level Guide

Task Model Effort
Simple fix Sol Low or Medium
New feature Sol High
Hard problem Sol Extra High
Fast scan Luna Extra High
Never Ultra —

Default for most developers: High. It's plenty capable.

Daily driver recommendation from Codex team: Sol Medium.

Stop Points Pattern

Tell the model exactly when to stop. Not vague. Not "use your judgment." Explicit checkpoints. Clear handoffs.

Planning tasks: 


Build this new feature.
Start by writing a plan only.
When the plan is done, stop and ask for my feedback.
Do not write any code yet.

Implementation tasks: 


The plan looks great. Build it out now.
Use computer use to test as you go.
Keep going until tests pass and you're satisfied.
Open a PR when done.
Stop after the PR is open. I'll handle reviews from there.

Debugging: 


Fix the bug in auth.ts.
Read the file first. Write your diagnosis.
Stop and show me the diagnosis before making changes.
I'll confirm before you proceed.

Benchmark Summary

Comparison Result
Sol Extra High vs Max 58 vs 59 pts. One point. 3x cheaper.
Sol Medium vs Fable 5 Sol Medium wins by 11.4 pts. 4x lower cost.
Luna Extra High vs Terra Same perf. 1.3x faster. 2.5x cheaper.
Ultra vs Extra High 3.1 extra pts. 3x more expensive.

Pattern: one tier down = same quality, dramatically cheaper.

Before and After

Before:

  • Sol Ultra running everything
  • Subagents cloning parent model
  • 3 subagents = 3 Sol Ultra instances burning simultaneously
  • 5-hour window gone in 90 minutes

After:

  • Sol Extra High as orchestrator
  • Sol Medium for implementation
  • Luna Extra High for scanning
  • max_depth = 1 stopping recursive spawning
  • AGENTS.md telling Sol not to spawn agents automatically
  • Stop points in every prompt

Result: 48 hours of continuous work. Zero limit hits. Same $200 plan.

Setup Checklist

1. Open AGENTS.md. Add: "Only spawn subagents when I explicitly ask you to." 2. Create 3 agent TOML files under ~/.codex/agents/ 3. Set max_depth = 1 and max_threads = 6 in config.toml 4. Switch default effort to Sol High (not Ultra) 5. Add explicit stop points to every long-running prompt
