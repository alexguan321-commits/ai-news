1. 🟠 [planning-with-files](https://github.com/OthmanAdi/planning-with-files) — Agent 持久化规划的「缺失拼图」

技术亮点： 把 AI coding agent 的计划持久化为 markdown 文件，crash-proof，支持跨 session 恢复。这解决了当前 AI coding agent 最大的痛点——每次对话都是从零开始，没有长期记忆。

为什么重要： 这直接对应了 Loop Engineering 的「State」组件。当前 Claude Code、Cursor 的致命缺陷是：agent 的计划只存在于当前 session，一旦中断就全部丢失。这个项目用「文件即状态」的朴素方案解决了这个问题——不需要向量数据库，不需要复杂的 RAG，一个 markdown 文件就够了。

因果链：

  • 当前 AI coding agent 的「失忆症」→ 每次都要重新理解项目上下文 → 浪费 30-50% 的 token 在「重新解释」上
  • planning-with-files 让计划持久化 → 下次运行直接读取 → token 效率提升 2x，上下文质量提升 3x
  • 12 个月内，所有主流 AI coding 工具都会内置类似功能——「持久化计划」会成为标配，不是差异化特性

策略建议：

  • 立即行动：如果你在用 Claude Code / Cursor，手动建立 `plans/` 目录，让 agent 每次运行前先读取上次的计划文件
  • Q3 前：评估 planning-with-files 是否适合你的工作流，特别是长周期项目（>1 周）
  • 18 个月内：关注这个领域是否会诞生「Agent State Management」SaaS 产品
