🔴 阿里巴巴将 Claude Code 列为高风险软件，禁止员工使用

事实： 据 TechCrunch 报道，阿里巴巴已将 Anthropic 的 Claude Code 列为「高风险软件」，禁止员工在公司环境中使用。这是中国科技巨头首次公开将一款 AI 编程工具定性为安全威胁。

判断： 这不是一个孤立事件，而是AI 工具地缘政治化的标志性时刻。因果链：Claude Code 需要访问代码库 + 可执行任意命令 → 数据外泄风险（即使是无意的）→ 在中美科技脱钩背景下，中国大厂无法容忍「美国 AI 工具接触核心代码」→ 这会引发连锁反应：字节、腾讯、华为会在 6 个月内出台类似禁令 → 国产 AI 编程工具（通义灵码、CodeGeeX）获得政策性市场保护。

更深层的含义： Anthropic 一直标榜「AI 安全」，但 Claude Code 的架构设计（需要完整代码库上下文 + 终端访问权限）本身就构成了供应链安全风险。这不是 Anthropic 的问题——所有需要深度代码访问的 AI 工具都面临同样的困境。

大胆预判：

  • 3 个月内，中国前 10 互联网公司中至少 7 家会发布类似的 AI 编程工具使用限制
  • 12 个月内，这会催生一个价值 $500M+ 的「国产 AI 编程工具」市场，通义灵码用户量会突破 1000 万
  • Anthropic 会失去中国市场——不是因为他们做错了什么，而是因为地缘政治不允许
  • 置信度：85%

🟠 Supabase MCP 可泄露整个 SQL 数据库——MCP 协议的安全债务集中爆发

事实： General Analysis 团队发现 Supabase 的 MCP（Model Context Protocol）实现存在严重安全漏洞，可泄露整个 SQL 数据库。这篇博文在 Hacker News 获得 848 点，470 条评论——是今日讨论最热烈的话题。与此同时，HN 上 MCP 相关帖子霸榜：「MCP: An (Accidentally) Universal Plugin System」（808 点）、「OpenAI adds MCP support to Agents SDK」（807 点）、「Claude Skills are awesome, maybe a bigger deal than MCP」（738 点）、「The 'S' in MCP Stands for Security」（730 点）、「A critical look at MCP」（623 点）。

判断： 这是MCP 协议的「Log4j 时刻」。因果链：MCP 设计初衷是让 AI 模型能调用外部工具 → 为了实现通用性，MCP 采用了宽松的权限模型 → 攻击面指数级扩大（每个 MCP server 都是一个潜在的数据泄露通道）→ Supabase 漏洞证明这不是理论风险 → 企业会重新评估 MCP 部署策略 → MCP 协议需要在「易用性」和「安全性」之间重新平衡。

关键洞察： MCP 的成功恰恰是它的安全隐患。它之所以被广泛采用（OpenAI、Anthropic、Chrome DevTools 都支持了），是因为它足够简单、足够通用——但这也意味着它没有内置细粒度的权限控制。「 accidentally universal」和「accidentally insecure」是同一枚硬币的两面。

大胆预判：

  • 6 个月内，会出现至少 3 起企业级 MCP 数据泄露事件登上新闻头条
  • 12 个月内，MCP 协议会强制引入 OAuth 2.0 式的权限 scopes，不兼容的 server 会被主流客户端拒绝
  • Anthropic 和 OpenAI 会联合发布「MCP Security Best Practices」——但为时已晚，损害已经造成
  • 置信度：90%

「Clever Hans」隐喻的深层含义

Computerphile 的视频用 Clever Hans（一匹被认为会算数的马，实际上是通过观察人的微表情来「回答」）来比喻 AI。这个隐喻在 2026 年特别有意义：

当我们在 benchmark 上看到 AI 模型的「惊人表现」时，我们是否也在像 Clever Hans 的观察者们一样，把自己的期望投射到模型上？

Agentic AI 让这个问题更尖锐。一个 Agent 完成了任务——它是真的「理解」了任务，还是只是学会了「读人」（读 prompt、读评估标准）？

没有答案，但这个问题值得每个人思考。
