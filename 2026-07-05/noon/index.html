📌 今日最重要的一件事

🔴 Alphabet 宣布 800 亿美元股权融资，巴菲特亲自下场 100 亿押注 AI 基础设施

🔗 来源：Alphabet IR | 来源：TechCrunch | 来源：CNBC | 来源：Reuters

发生了什么： Alphabet（Google 母公司）宣布计划通过股权融资筹集 800 亿美元，用于「扩展 AI 基础设施和全球算力」。这是美国科技史上最大规模的股权融资之一。

关键数字拆解：

  • Berkshire Hathaway 私人配售 100 亿美元：其中 50 亿美元买入 Class A 普通股（每股 $351.81），50 亿美元买入 Class C 资本股（每股 $348.20），均低于当日收盘价——这是巴菲特给 Alphabet 的「打折背书」
  • 承销发行 300 亿美元：包括 150 亿美元的「强制转换优先股存托份额」（一种混合证券，既有债券的固定收益又有股票的上涨空间）
  • ATM 市场发行 400 亿美元：通过 Class A 和 Class C 股票的「按市价发行」计划，预计 Q3 开始
  • 2026 年资本支出预算已上调至 1800-1900 亿美元（此前预估 1750-1850 亿），这个数字本身就超过了大多数国家的 GDP

Alphabet 官方声明的关键句： "The company is experiencing strong demand for its AI solutions and services from enterprises and consumers, at levels that are exceeding the company's available supply."——翻译：需求已经超过供给能力，不是不想赚更多钱，是算力不够。

为什么这是今天最重要的事：

这不是一次普通的融资。这是 AI 军备竞赛的分水岭事件，标志着三个根本性转变：

第一，「巴菲特效应」的信号意义。 巴菲特 94 岁，Berkshire 持有超过 2600 亿美元现金储备，一直因不愿追高而被市场称道。现在他拿出 100 亿投 Alphabet，而且是以折价买入——这不是财务投资，这是战略判断。巴菲特在说：AI 基础设施的需求增长是真实的，不是泡沫。当世界上最谨慎的资本配置者下注 AI 基础设施，整个市场的风险定价逻辑都会改变。

第二，股权融资而非债务融资，说明 Alphabet 对 AI 回报的时间框架是「长周期」。 如果是短期能回本的项目，用债务就够了。选择股权稀释，意味着 Alphabet 预期 AI 基础设施的投资回报周期在 5-10 年以上，但回报倍数足以覆盖稀释。这和 Anthropic ARR 从 10 亿到 450 亿只用了不到三年的增长曲线一致——AI 基础设施的需求不是线性增长，是指数级的。

第三，800 亿股权融资 + 1800-1900 亿资本支出 = Alphabet 正在把自己变成一家「算力公用事业公司」。 Sundar Pichai 在 Google I/O 上说的不是空话：他们真的在按照电力公司的逻辑来建设 AI 基础设施——大规模、长周期、高壁垒、寡头垄断。

因果链分析： 


Alphabet 800亿股权融资 + 1800-1900亿资本支出
  → 2026-2027年大规模采购 GPU/TPU（主要受益者：NVIDIA、Broadcom、Marvell）
    → 数据中心建设加速（主要受益者：Quanta、Vertiv、Eaton）
      → AI 算力供给增加 → Gemini 模型能力跃升 → Google Cloud 企业客户增长
        → 与 AWS（Amazon）和 Azure（Microsoft）的三巨头竞争白热化
          → 其他巨头被迫跟进更大规模融资 → 整个科技板块进入「AI 基础设施超级周期」
            → 18个月内，AI 算力成为像石油一样的战略资源，供给瓶颈推动价格持续上涨

大胆预判：

1. 12 个月内，Meta 和 Microsoft 将各自宣布超过 500 亿美元的类似股权融资计划，因为 Alphabet 的举动重新定义了「合理融资规模」的锚点。置信度：85%。 2. 18 个月内，NVIDIA 的年收入将突破 3000 亿美元（当前约 1300 亿），因为仅 Alphabet 一家的资本支出就接近 1900 亿。置信度：75%。 3. 6 个月内，Berkshire Hathaway 将追加对 AI 基础设施的投资（可能再投 50-100 亿），因为巴菲特一旦确认方向就不会浅尝辄止。置信度：60%。 4. 24 个月内，Alphabet 的 AI 基础设施收入（Google Cloud + AI 服务）将超过 AWS 成为云计算市场第一，因为他们在 TPU 自研芯片上的成本优势会在规模化后体现。置信度：55%。

策略建议：

  • 对 AI 创业者：Alphabet 的算力扩张意味着 API 价格在 12-18 个月内会下降 40-60%（供给增加 + 竞争加剧）。现在基于 Gemini API 构建的产品，到 2027 年成本会大幅降低。建议在 Q3 开始评估 Gemini API 作为主力模型，不要只绑定 OpenAI/Anthropic。
  • 对投资者：关注 Alphabet 供应链上的「卖铲人」——Broadcom（定制芯片）、Vertiv（数据中心散热）、Eaton（电力管理）。这些公司会直接受益于 1800 亿资本支出的落地。
  • 对 FDE 团队：当算力变成「水电煤」，企业 IT 的核心竞争力不再是「有没有 GPU」，而是「能不能把 AI 嵌入业务流程」。这正是 FDE 模式的价值所在。

🔴 HN 社区集体「反 LangChain」：AI Agent 框架的范式转移已经完成

🔗 来源：Why we no longer use LangChain (480 pts) | 来源：Langchain Is Pointless (386 pts) | 来源：The Problem with LangChain (268 pts) | 来源：Re-implementing LangChain in 100 lines (252 pts)

发生了什么： Hacker News 首页同时出现了 6 篇以上关于 LangChain 的帖子，总点数超过 2000。核心观点高度一致：LangChain 的抽象层太厚、调试困难、在生产环境中不可靠。社区正在转向「直接用 LLM API + 少量胶水代码」的极简方案。

深度分析：

这不是 LangChain 一家公司的问题。这是整个「AI 中间件」赛道的范式转移。

第一阶段（2023 年初）： LLM API 刚出来，大家不知道怎么编排多步调用。LangChain 提供了 Chain、Agent、Memory 等抽象，降低了入门门槛。此时 LangChain 的价值是「教育」——让开发者理解 LLM 应用的可能性。

第二阶段（2023 年中 - 2024 年）： 开发者开始把 LangChain 用于生产环境。问题暴露：抽象层太厚导致调试困难（「5 层抽象只为了改一个细节」），回调机制不透明，版本升级频繁破坏兼容性。Octomind 的文章精准描述了这个问题：「LangChain does not provide a method for externally observing an agent's state」——你甚至无法从外部观察 agent 在干什么。

第三阶段（2024 年中 - 2026 年）： LLM 模型本身的能力大幅提升（context window 从 4K 到 2M，工具调用能力内建），使得大量 LangChain 的抽象变得不必要。你不再需要一个框架来管理 memory——模型自己就能处理 200K token 的上下文。你不再需要一个框架来编排工具——模型原生支持 function calling。

因果链： 


LLM 原生能力提升（长上下文 + 内建工具调用 + 更好的指令遵循）
  → LangChain 的抽象层从「必要」变成「累赘」
    → 开发者转向「LLM API + 100 行胶水代码」方案
      → AI Agent 框架赛道重新洗牌
        → 赢家不再是「提供最多抽象的框架」，而是「提供最好可观测性和可靠性的基础设施」
          → LangSmith（LangChain 的可观测性产品）可能比 LangChain 本身更有价值

与 LLM Wiki 的关联： 这直接验证了知识库中「Loop Engineering」文章的核心观点——Agent 的关键不是框架，而是「Closed Loop + 可观测性 + Memory 在对话之外」。LangChain 失败的原因恰恰是它在这三点上都做得不够好。

大胆预判：

1. 12 个月内，LangChain 的 GitHub star 增长将停滞，企业级 agent 项目中 LangChain 的使用率从当前的约 40% 下降到 20% 以下。置信度：80%。 2. 18 个月内，AI agent 框架赛道将出现新的领导者，最可能是 Anthropic 的 Claude Code + Computer Use 生态，或 OpenAI 的 Assistants API——因为它们把框架能力和模型能力绑定在一起，消除了「抽象层过厚」的问题。置信度：65%。 3. 对创业者的影响：如果你现在还在用 LangChain 构建 agent，建议在 Q3 开始评估替代方案。不是因为它不能用，而是因为你的竞争对手正在用更简单、更可控的方案构建更好的产品。

🟠 Gemini 3.5 Pro 泄露：2M token 上下文 + 超越 Claude Fable 5 的推理能力

🔗 来源：YouTube - Gemini 3.5 Pro LEAKS | 来源：X 泄露帖 | 来源：Coursiv 分析 | 来源：Business Insider

发生了什么： Google 即将发布的 Gemini 3.5 Pro 模型通过泄露的内部基准测试结果曝光。关键规格：

  • 2M token 上下文窗口（在 Deep Think 推理模式下）——这是目前所有模型中最大的
  • 在多个零样本推理任务上超越 Anthropic Claude Fable 5——注意这些是内部数据，尚未经过独立验证
  • 预计 2026 年 7 月中旬发布——从 Google I/O 2026（5 月 20 日）首次预览后，经历了两次延期（原定 6 月，推迟到 7 月）

深度分析：

2M token 上下文的战略意义： 这不是一个数字游戏。2M token 意味着你可以一次性把整个代码库（约 100-200 万 token）塞进上下文窗口，让模型理解整个系统的架构、依赖关系和调用链。这对 AI 编码工具（如 Claude Code、Cursor）是降维打击——当前 Claude 的上下文窗口是 200K token，差了一个数量级。

但要注意： 上下文窗口大不等于用得好。Google 之前的 Gemini 1.5 Pro 也有 1M/2M 上下文窗口，但在实际使用中，超过 100K token 后的信息检索准确率显著下降（所谓的「lost in the middle」问题）。Gemini 3.5 Pro 是否解决了这个问题，需要等独立基准测试。

竞争格局影响： 如果 Gemini 3.5 Pro 真的在推理能力上超越 Claude Fable 5，这将是 Google 自 Gemini 系列发布以来第一次在旗舰模型能力上领先。考虑到 Google 在 I/O 上展示的 Gemini 多模态能力、与 Google Search/Workspace 的深度集成，以及今天公布的 800 亿融资支撑的算力扩张——Google 正在形成一个「模型 + 产品 + 基础设施」的完整闭环。

因果链： 


Gemini 3.5 Pro 发布（2M 上下文 + 强推理）
  → Google Cloud 企业客户获得「大上下文 + 强推理」的独特组合
    → 长文档分析、代码库理解、多轮复杂对话场景被解锁
      → Anthropic/OpenAI 被迫加速下一代模型的发布
        → 模型能力竞争从「benchmark 分数」转向「实际工作场景的完成质量」
          → 对开发者来说，「多模型策略」成为必须——不同任务用不同模型

大胆预判：

1. 3 个月内（2026 年 10 月前），Gemini 3.5 Pro 将公开发布并开放 API。Google 不会让它停留在内部使用太久。置信度：90%。 2. 6 个月内，基于 2M 上下文窗口的「全代码库 AI 编码助手」产品将出现（可能是 Google 自己的 Gemini Code Assist 升级版），能理解整个 repo 的架构。置信度：70%。 3. 12 个月内，Anthropic 将发布 Claude 5.x 系列，上下文窗口至少扩展到 1M token，以应对 Gemini 的竞争。置信度：80%。

LangChain 的衰落揭示了一个更深层的趋势：随着 LLM 能力提升，中间件的价值在缩水。

  • LLM 原生支持 function calling → 不需要框架来管理工具调用
  • LLM 的 context window 达到 2M → 不需要框架来管理 memory
  • LLM 的指令遵循能力增强 → 不需要框架来编排多步推理

推论： 未来 AI 应用栈的中间层会越来越薄。价值集中在两端：底层是模型（OpenAI/Anthropic/Google），顶层是应用（直接解决用户问题的产品）。中间件要么变得极轻（100 行胶水代码），要么提供模型无法替代的能力（可观测性、合规审计、安全护栏）。
