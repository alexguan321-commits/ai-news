🔬 深度分析

AI Agent 安全的"信任链"问题

Claude Code 被劫持事件揭示了一个根本问题：AI agent 的信任链是断裂的。

传统软件的安全模型是"零信任"——每个组件都不信任其他组件，所有访问都需要验证。但 AI agent 的设计哲学是"高信任"——agent 需要访问工具、数据、API 才能完成任务，每次访问都验证会严重影响效率。

这创造了一个悖论：agent 越强大，安全风险越高；安全限制越多，agent 越弱。

解决方案不是"更少的信任"或"更多的信任"，而是分层的信任： 1. Agent 可以访问只读数据 2. Agent 可以执行低风险操作（创建分支、提交 PR） 3. Agent 不能执行高风险操作（合并到 main、访问生产环境） 4. 所有操作都有审计日志，异常行为触发告警
